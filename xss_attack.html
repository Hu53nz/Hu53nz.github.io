What is a Cross-Site Scripting (XSS) Attack?

An XSS attack occurs when an attacker injects malicious scripts into content from otherwise trusted websites.

Examples of XSS Attacks

  • Stored XSS: Malicious script is stored on the server and executed when users visit the site.
  • Reflected XSS: Malicious script is part of the URL and executed immediately.

Before and After Input Sanitization

Before Input Sanitization

  • Test Case 1: <script>alert('XSS')</script>
    Bot Response: <script>alert('XSS')</script>

    Explanation: This output is dangerous because the script tag is executed, triggering the JavaScript code and potentially compromising user data or control over the website.

  • Test Case 2: '; DROP TABLE users;--
    Bot Response: '; DROP TABLE users;--

    Explanation: This input can lead to a SQL injection attack, where the attacker can delete the user table or perform other malicious database operations.

  • Test Case 3: ; ls
    Bot Response: ; ls.asm

    Explanation: This input may allow the attacker to execute system commands and list sensitive files on the server.

  • Test Case 4: && cat /etc/passwd
    Bot Response: && cat /etc/passwd | grep -v root ...

    Explanation: This input can exploit shell injection vulnerabilities, potentially allowing access to sensitive files like /etc/passwd on Unix-based systems.

  • Test Case 5: <img src='x' onerror='alert(1)'>
    Bot Response: <img src='x' onerror='alert(1)'>

    Explanation: This could trigger an image error event, running arbitrary JavaScript that could harm the system or compromise user data.

Solutions to Prevent Insecure Outputs

1. HTML Character Escaping (html.escape)

  • Transforms characters like `<`, `>`, `&` into their HTML-encoded equivalents (`<`, `>`, `&`) to prevent them from being interpreted as code.

The html.escape() function converts special characters in the output string (e.g., <, >, &) into HTML-safe representations (<, >, &), preventing XSS attacks by rendering potentially dangerous code as text.

2. Regular Expression Filtering (re.sub())

  • Uses regular expressions to filter out potentially harmful patterns such as script tags and on-event handlers.

The re.sub() function identifies harmful command phrases like `exec`, `cmd`, or `.sh` extensions and replaces them, protecting against command injections.

3. Input Validation

  • Validates user inputs against a whitelist of acceptable values to ensure that only expected data is processed by the application.
