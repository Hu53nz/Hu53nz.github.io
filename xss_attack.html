What is a Cross-Site Scripting (XSS) Attack?

An XSS attack occurs when an attacker injects malicious scripts into content from otherwise trusted websites.

Pre-sanitization

Before Input Sanitization

User input directly rendered without checks:

                            <script>
                                alert("Hacked!");
                            </script>

After Input Sanitization

Sanitized input prevents script execution:

                            <!-- <script> becomes harmless text -->
                            <span>&lt;script&gt;alert("Hacked!");&lt;/script&gt;</span>

Explanation: The input is sanitized by escaping harmful characters (e.g., <, >, and ").

Solutions to Prevent XSS Attacks

  • Escape user input using HTML character encoding.
  • Validate input to allow only safe content.
  • Use Content Security Policy (CSP) to restrict script execution.
